1. Deployment Overview
This document defines the deployment strategy for the AI Schema Builder application across three environments: Development, Staging, and Production. The strategy follows a containerized microservices approach using Docker and orchestrated via Kubernetes (AWS EKS) or AWS ECS Fargate.

2. Infrastructure Architecture
2.1 AWS Production Architecture
Route 53 — DNS with health checks and failover routing
CloudFront CDN — caches static frontend assets globally
Application Load Balancer (ALB) — distributes traffic to backend services
EKS (Elastic Kubernetes Service) — backend API pod orchestration
RDS PostgreSQL (Multi-AZ) — primary database with automated failover
ElastiCache Redis — session store and API response caching
S3 — static frontend hosting and schema export file storage
Secrets Manager — encrypted storage for Gemini API keys and DB credentials
CloudWatch — metrics, logs, alarms, and dashboards
2.2 Container Architecture
Each service is packaged as a Docker container. Images are tagged with the Git commit SHA and pushed to Amazon ECR (Elastic Container Registry).

3.2 GitHub Actions Workflow
The following key pipeline steps are executed on every deployment:
1. Checkout code
2. Run: npm test && npm run build
3. docker build -t $ECR_REPO:$GIT_SHA .
4. docker push $ECR_REPO:$GIT_SHA
5. kubectl set image deployment/api api=$ECR_REPO:$GIT_SHA
6. kubectl rollout status deployment/api
7. Run smoke tests → rollback on failure

5. Database Strategy
5.1 Migrations
Database migrations are managed using Knex.js migrations. Migrations run automatically at application startup in development and are run as a pre-deploy step (Kubernetes Job) in production.
5.2 Backup & Recovery
RDS automated backups — 7-day retention with point-in-time recovery
Daily snapshots exported to S3 (cross-region for disaster recovery)
Recovery Time Objective (RTO): < 1 hour
Recovery Point Objective (RPO): < 15 minutes

7. Monitoring & Alerting
7.1 Key Metrics
API response time (p50, p95, p99) — alert if p95 > 2s
Error rate — alert if > 1% of requests return 5xx
Gemini API call success rate — alert if < 98%
Database connection pool utilization — alert if > 80%
Pod/container restart count — alert on > 3 restarts/hour
7.2 Alerting Channels
PagerDuty — for P0/P1 production incidents
Slack #alerts — for warnings and non-critical events
Email — daily digest of system health metrics
